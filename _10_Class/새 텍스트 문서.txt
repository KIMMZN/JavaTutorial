클래스는 객체를 만들기 위해 정의한것
클래스 : 멤버변수 + 메서드로 정의
클래스로부터 만든 객체는 서로 독립적
객체를 참조하는 변수는 원시변수가 아닌 참조변수
참조변수는 주소를 저장, 객체는 자바의 힙영역에 저장
참조변수가 저장하고 있는 주소값을 참조하여 객체에 접근
new 연산자로 객체를 만들 때는 생성자로 만든다.
생성자는 객체를 만들 때 처음 호출되는 메서드이다.
  -생성자는 클래스 이름과 동일하다. -

원시변수 : 실제 사용하려는 값, 실제 값이라 표현
참조변수는 실제 사용하려는 객체의 주소를 저장, ; 객체를 참조하기 위한 주소 ;
*
소문자로 시작하는건 원시변수 int 등등..
대문자로 시작하는건 참조변수 random stirng

변수는 원시변수 , 참조변수로 구성
원시 변수는 실제 값
참조 변수는 객체를 참조하기 위한 주소;

**

변수는 값을 저장하는 공간이다
값은 메모리에 저장한다
공유는 자원의 낭비를 줄인다.
공유는 메모리의 낭비를 줄인다.(메모리 사용양을 줄일 수 있다.)

int는 4바이트


클래스 : 레시피
멤버변수 : 재료
메서드 : 조리과정
객체 : 음식


--1
레시피는 실제 사람이 먹을 음식이라고 할수 있을까?
1. 아니다. 왜냐면 실제 존재하지 않기 때문에 사람이 먹을 음식이라고 할 수 없다.

레시피는 왜 만들었나
: 재사용성을 높히기 위해서
2. 음식을 누구나 만들수 있게 하기위해 레시피가 존재함.


레시피와 실제 만들어진 음식은 어떤 차이가 있을까?

3. 실제 음식은 실제로 존재해 먹을수 있고, 레시피는 만드는 방법만 있어 먹을수 없다.

--2

레시피는 1개만 있고 레시피를 통해 만들어진 음식도 1개라고 말 할수 있을까 ?
*아니다 레시피를 통해서 많은 음식을 만들 수 있따.

레시피가 있다면 음식도 만들어 진 것일까 ?
*아니다 음식을 만들어야 만들어진 것이다.

서로 다른 쭈꾸미 레시피로 각각 요리를 했다면 음식은 쭈꾸미라는 성격을 갖는다.
이 음식을 서로 같은 음식이라고 할수 있을까 ?
*아니다

--3
레시피는 여러 사람과 공유할 수 있다, 이때 레시피는 하나이고 레시피를 참조하는 사람도 1명 일까?
*아니다 레시피는 하나이지만 참조하는 사람은 한명일수도 있고 여러명일수 있다. 
* : 공유는 자원의 낭비를 줄인다.; || 메모리의 낭비를 줄인다.

여러 사람이 하나의 레시피를 참조 할때, 어떤 사람이 레시피를 수정한다면 다른 사람은 수정된 레시피를 볼까? 아니 수정되지 않은 상태의 레시피를 볼까?
*수정하고 저장한다면 수정된 상태의 레시피를 본다
*수정하고 저장하지 않는다면 수정되지 않은 상태의 레시피를 본다.
